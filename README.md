# Ascend

> **Break the Scroll. Earn Your Feed.**

Ascend is an Android app that helps you reduce screen time **without** the cold-turkey app blocks that make people frustrated and uninstall. When you hit a daily limit on a monitored app, Ascend doesn't lock you out — it shows you a **question**. Answer it to earn a short grace period. Each round the question gets **harder**, until the effort outweighs the urge to keep scrolling.

The core insight from the product blueprint: *addiction can't be removed instantly — sudden hard blocks cause frustration and app deletion.* So Ascend replaces the hard block with a **Cognitive Friction Ladder**.

> ⚠️ **Status: work in progress.** The app is being built milestone by milestone (see [Roadmap](#roadmap)). It currently runs as an Expo **development build** on a real Android device.

---

## How it works (the core mechanic)

1. You pick apps to monitor (Instagram, YouTube, …) and set a **daily time limit** per app.
2. Ascend tracks real foreground usage via Android's `UsageStatsManager`.
3. When a limit is reached, a **friction overlay** appears over the blocked app.
4. To earn a **grace period** (5/10/15 min) you must answer a question (Math / Trivia / Logic / Typing).
5. Each subsequent round the **difficulty climbs** and never resets until midnight.
6. A wrong answer → retry, or **skip** (penalty: +1 level, no grace). Or tap **"I'm done for today"** to block the app for the rest of the day.

There's also a **baseline** (your average usage over the *past* 7 days, computed once on first launch), a **Dashboard**, **Stats**, a **Leaderboard** (% screen-time reduction), and **Settings**.

## Architecture in one picture

Ascend is a React Native (Expo) app with a small **Kotlin native module**. The split is deliberate:

```
  ┌─────────────────────────────────────────────┐
  │  JavaScript / TypeScript  =  the "brain"     │
  │  • all UI (screens, components)              │
  │  • the friction-ladder engine (Zustand)      │
  │  • app config, question bank, stats logic    │
  └───────────────────────┬─────────────────────┘
                          │  Expo Modules bridge
  ┌───────────────────────┴─────────────────────┐
  │  Kotlin native module  =  "eyes + muscle"    │
  │  • read per-app usage (UsageStatsManager)    │
  │  • list installed apps (PackageManager)      │
  │  • check/open special permissions            │
  │  • (Phase D) overlay trigger + bg service    │
  └─────────────────────────────────────────────┘
```

Everything that *can* be done in JavaScript **is** — only the three things JS genuinely can't do (read usage, draw over other apps, run in the background) live in Kotlin.

## Tech stack

| Layer | Choice |
|---|---|
| App framework | **React Native** via **Expo** (SDK 54), TypeScript |
| Navigation | **Expo Router** (file-based) |
| State | **Zustand** (+ `persist` → AsyncStorage) |
| Native module | **Kotlin** via the **Expo Modules API** (`modules/ascend-native`) |
| Usage data | Android `UsageStatsManager` (event-based) |
| Design system | "Summit" palette, Bricolage Grotesque + Outfit fonts |
| Backend | *Deferred* — everything runs on-device for now (Node/Express + Postgres + Firebase planned) |

## Getting started

### Prerequisites
- **Node** 20+ and npm
- **JDK 17** (the Android build requires it — newer JDKs can fail)
- **Android SDK** + **NDK** + command-line tools (install via Android Studio's SDK Manager)
- A **physical Android device** with USB debugging — usage tracking returns almost nothing on emulators
- `ANDROID_HOME` and `JAVA_HOME` set in your shell

> **Expo Go will not work** — the app has custom native code, so it needs a development build.

### Run it
```bash
cd mobile
npm install
npx expo run:android      # builds the native app, installs on your device, starts Metro
```
The first build is slow (downloads the NDK + dependencies, ~15 min) and needs internet; subsequent builds are seconds. After the first build:
- **Editing JS/TS** → hot-reloads instantly over USB (no rebuild).
- **Editing Kotlin** → re-run `npx expo run:android` (a native rebuild).

To grant the two special permissions, follow the in-app onboarding (Usage Access + Display-over-other-apps), or toggle them in Android Settings.
